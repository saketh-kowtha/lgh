Configuration

lazyhub is zero-config out of the box. Everything here is optional.

Config file location

# macOS / Linux
~/.config/lazyhub/config.json

# Windows
%APPDATA%\lazyhub\config.json

Create the file if it doesn't exist — lazyhub uses sensible defaults for any missing keys.

Full example

{
  "panes": ["prs", "issues", "branches", "actions", "notifications"],
  "defaultPane": "prs"
}

Schema reference

KeyTypeDescriptionDefault
panes string[] Which panes to show in the sidebar (and in what order). Omit any pane to hide it completely. all panes
defaultPane string Which pane to open on startup. Must be one of the values in panes. "prs"

panes

Controls which panes appear in the sidebar. Valid values:

prs issues branches actions notifications

The order in the array determines the order in the sidebar and the Tab-cycling order.

Show only PRs and Issues

{
  "panes": ["prs", "issues"]
}

Show everything except notifications

{
  "panes": ["prs", "issues", "branches", "actions"]
}

Reorder: start with Issues

{
  "panes": ["issues", "prs", "branches", "actions", "notifications"],
  "defaultPane": "issues"
}
Tip: If panes contains an unrecognized value, lazyhub silently ignores it. If panes is empty or missing, all panes are shown.

defaultPane

Sets which pane is active when lazyhub starts. Must be one of the values in your panes array (or a valid pane name if panes is omitted).

{
  "defaultPane": "issues"
}

If defaultPane is not in the visible panes list, lazyhub falls back to the first visible pane.

Editing the config live

Changes to config.json take effect on the next launch. There is no hot-reload — exit lazyhub (q) and re-open.

# Quick edit
$EDITOR ~/.config/lazyhub/config.json

# Create the directory if it doesn't exist
mkdir -p ~/.config/lazyhub && $EDITOR ~/.config/lazyhub/config.json

Validation

lazyhub validates the config on startup:

  • Invalid JSON → prints a parse error, uses defaults, continues
  • Unknown keys → silently ignored (forward-compatible)
  • Invalid pane names → silently ignored
  • Empty panes array → falls back to showing all panes

Future config options

These are planned for a future release and are not yet supported:

  • theme — custom color palette
  • cache.ttl — override the 30-second data cache TTL
  • repo — pin a default repo (instead of auto-detect)
  • editor — override $EDITOR for compose views

Open an issue if you want to request a specific config option.
